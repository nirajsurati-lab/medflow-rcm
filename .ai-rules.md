# Project AI Rules

## Tech Stack

- Frontend: Next.js (App Router) + TypeScript
- Backend: Supabase (Auth, Database, Storage)
- Styling: TailwindCSS
- State: React hooks only unless necessary
- API: Supabase client

## General Rules

- Always use TypeScript.
- No `any` types.
- Prefer functional React components.
- Use async/await instead of promises.

## Next.js Rules

- Use App Router (`/app` directory).
- Server components by default.
- Client components only when required (`"use client"`).
- Use server actions when interacting with Supabase when possible.

## Supabase Rules

- Use `@supabase/supabase-js`.
- Create a single Supabase client in `/lib/supabaseClient.ts`.
- Do not duplicate Supabase initialization.
- Use typed queries where possible.

## Folder Structure

app/
components/
lib/
hooks/
types/
utils/

## Component Rules

- One component per file.
- Components must be reusable.
- Use Tailwind for styling.
- Avoid inline styles.

## Data Access Rules

- All Supabase queries go through `/lib`.
- No Supabase calls directly inside UI components.

## Error Handling

- Handle all async calls with try/catch.
- Show user-friendly errors.

## Performance

- Use server components for data fetching.
- Avoid unnecessary client-side state.

## Security

- Never expose Supabase service role keys.
- Use environment variables.
